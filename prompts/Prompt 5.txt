## Objetivo da fase

Criar um serviço backend seguro para comunicação com a API do Asaas em sandbox, sem ainda expor checkout completo ao usuário.

## Escopo obrigatório

1. Criar módulo `integrations/asaas`.
2. Criar `AsaasService`.
3. Criar cliente HTTP configurado por `.env`:

   * `ASAAS_API_URL`;
   * `ASAAS_API_KEY`;
   * `ASAAS_ENVIRONMENT=sandbox`;
   * timeouts;
   * retries moderados;
   * logs sanitizados.
4. Implementar métodos:

   * criar cliente;
   * atualizar cliente;
   * buscar cliente;
   * criar cobrança de cartão;
   * criar cobrança de boleto;
   * criar assinatura;
   * buscar assinatura;
   * cancelar assinatura;
   * buscar pagamento.
5. Criar DTOs internos.
6. Criar tratamento de erros padronizado.
7. Criar testes com mocks HTTP.
8. Opcionalmente criar comando/script interno de teste sandbox, sem expor rota pública sensível.

## Correção obrigatória

Não criar método para Pix avulso único via Asaas.

Não criar fallback automático que envie Pix único para Asaas.

Pix único avulso pertence ao fluxo próprio BR Code/EMV, que será implementado em fase específica.

## O que não implementar agora

Não implementar checkout público, webhooks, assinaturas reais completas, telas, Pix BR Code, confirmação manual de Pix, portal de transparência ou área do doador.

## Segurança obrigatória

1. Nunca expor `ASAAS_API_KEY` no frontend.
2. Não logar headers de autorização.
3. Não logar dados completos de cartão.
4. Sanitizar payloads nos logs.
5. Todas as chamadas devem passar pelo backend.
6. Variáveis reais devem estar apenas em `.env`, nunca em arquivos versionados.
7. `.env.example` deve conter apenas placeholders.

## Testes obrigatórios

1. Teste de criação de cliente com mock.
2. Teste de criação de cobrança cartão com mock.
3. Teste de criação de boleto com mock.
4. Teste de criação de assinatura com mock.
5. Teste de erro 401 do Asaas.
6. Teste de timeout.
7. Teste de payload inválido.
8. Se possível, teste manual em sandbox com dados fictícios.
9. Garantir que nenhum fluxo de Pix avulso chama Asaas.

## Critérios de aceite

1. `AsaasService` centralizado.
2. Variáveis de ambiente documentadas.
3. Testes passam.
4. Nenhuma chave exposta.
5. Nenhum fluxo de Pix avulso usa Asaas.
6. Código preparado para ser consumido pelas fases de cartão, boleto e assinaturas.
