## Objetivo da fase

Criar cadastro administrativo de animais, galeria de imagens e páginas públicas para apresentar animais acolhidos, em tratamento, disponíveis para adoção ou adotados.

## Escopo obrigatório

1. Criar páginas admin:

   * `/admin/animais`;
   * `/admin/animais/novo`;
   * `/admin/animais/:id`.
2. Criar páginas públicas:

   * `/animais`;
   * `/animais/:slug` ou `/animais/:id`.
3. Criar endpoints:

   * `GET /api/v1/public/animals`;
   * `GET /api/v1/public/animals/:id`;
   * CRUD admin de animais;
   * upload de imagens;
   * ordenação de galeria.
4. Campos:

   * nome;
   * espécie;
   * sexo;
   * idade aproximada;
   * porte;
   * status;
   * história;
   * necessidades;
   * foto principal;
   * galeria;
   * público/interno;
   * observações internas.
5. Filtros públicos:

   * espécie;
   * porte;
   * status.
6. CTAs:

   * quero ajudar;
   * tenho interesse em adoção;
   * doar para animais como este.
7. Formulário simples de interesse em adoção/contato, protegido contra spam.

## Correções obrigatórias

1. Não expor observações internas.
2. Imagens devem ser otimizadas.
3. Não usar fotos enormes sem tratamento.
4. Não implementar campanhas emergenciais completas nesta fase.
5. Não misturar cadastro de animal com fluxo financeiro.
6. Usar MySQL via Prisma.

## Segurança obrigatória

1. Upload seguro.
2. Proteção contra spam no formulário.
3. RBAC para admin.
4. Auditoria em alterações relevantes.
5. Validar mime e tamanho de imagens.
6. Não permitir upload de script disfarçado de imagem.

## Testes obrigatórios

1. Criar animal.
2. Editar animal.
3. Publicar/despublicar.
4. Upload de múltiplas fotos.
5. Página pública lista somente públicos.
6. Filtros funcionam.
7. Mobile não quebra.
8. Dados internos ficam ocultos.
9. Persistência no MySQL funciona.

## Critérios de aceite

1. Animal cadastrado aparece no site.
2. Galeria funciona.
3. Dados internos ficam protegidos.
4. Build e testes passam.
5. Páginas públicas são responsivas e atraentes.
