Objetivo da fase

Criar a base inicial profissional do projeto em monorepo com Turborepo, pnpm, TypeScript, ESLint, Prettier, Docker local, apps separados e pacotes compartilhados, utilizando MySQL 8.0 local como banco relacional e Redis para filas/cache.

Escopo obrigatório
Analisar a estrutura atual do projeto.
Confirmar se o projeto está vazio ou parcialmente iniciado.
Criar estrutura monorepo com Turborepo e pnpm.
Criar obrigatoriamente:
apps/web — Next.js App Router para portal público e área do doador;
apps/admin — Next.js App Router para painel administrativo;
apps/api — NestJS com Fastify;
packages/ui;
packages/database;
packages/validators;
packages/types;
packages/config, se útil.
Criar pnpm-workspace.yaml.
Criar turbo.json.
Criar package.json raiz com scripts:
dev;
build;
lint;
test;
format;
typecheck.
Configurar TypeScript compartilhado.
Configurar ESLint e Prettier no root.
Configurar .gitignore.
Criar docker-compose.yml local com:
MySQL 8.0;
Redis.
Configurar o MySQL local com:
imagem: mysql:8.0;
usuário: root;
senha: CreativeStudio@2026;
database: lardosanjos;
porta: 3306;
charset/collation recomendado: utf8mb4.
Criar .env.example para:
root;
apps/api;
apps/web;
apps/admin.
Incluir nos .env.example a string:

Adicionar comentário explicando que o caractere @ da senha deve ser codificado como %40 na URL.
Criar endpoint básico /health no backend.
Criar página inicial temporária em apps/web.
Criar página inicial temporária em apps/admin.
Garantir que o projeto compile.
Correções obrigatórias desta fase
Não usar PostgreSQL.
Não configurar postgres, pg, pg_dump, postgresql:// ou qualquer provider PostgreSQL.
Usar MySQL 8.0 no Docker.
Criar também apps/admin, pois a arquitetura do plano prevê apps/web, apps/admin e apps/api.
O que não implementar agora

Não implemente migrations completas, autenticação, layouts finais, pagamentos, Pix, Asaas, área do doador, admin financeiro, animais, campanhas, PWA ou deploy.

Segurança mínima
Configurar CORS de forma restrita e parametrizada por .env.
Usar Helmet no backend.
Não criar secrets reais no código.
Não versionar .env.
Não tratar a senha local do MySQL como secret de produção; ela deve aparecer apenas em .env.example e Docker local conforme definido no plano.
