## Correções obrigatórias desta fase

1. Não usar PostgreSQL em produção.
2. Não usar `pg_dump`.
3. Não configurar container PostgreSQL.
4. Não usar connection string `postgresql://`.
5. Usar `DATABASE_URL` compatível com MySQL.
6. Usar migrations Prisma com provider MySQL.
7. Backups devem ser feitos com `mysqldump`.
8. O backup periódico deve rodar a cada 12 horas e enviar para o Cloudflare R2, com retenção de 30 dias, conforme o plano.
9. Documentar teste de restore do backup MySQL.
10. Não usar usuário `root` do MySQL em produção, salvo decisão explicitamente justificada e aprovada.
