## Objetivo da fase

Criar a carteirinha digital do doador, com número de membro, QR Code de validação pública, visual premium, download e compartilhamento.

## Escopo obrigatório

1. Criar página:

   * `/dashboard/carteirinha`.
2. Criar página pública:

   * `/anjo/validar/:cardNumber`;
   * ou rota equivalente.
3. Criar endpoints:

   * `GET /api/v1/donor/card`;
   * `POST /api/v1/donor/card/generate`;
   * `GET /api/v1/public/cards/:cardNumber`;
   * endpoint de download PNG/PDF, se backend for responsável.
4. Criar ou consultar `donor_cards` no MySQL.
5. Gerar número único de carteirinha.
6. Gerar QR Code com URL pública de validação.
7. Exibir:

   * logo;
   * nome público;
   * plano;
   * status;
   * data de início;
   * número de membro;
   * selos;
   * QR Code.
8. Permitir:

   * baixar PNG;
   * baixar PDF;
   * compartilhar WhatsApp;
   * copiar link.
9. Criar design com gradiente, bordas arredondadas e aparência profissional.

## Correções obrigatórias

1. QR Code público não pode expor CPF, endereço, e-mail, telefone ou dados sensíveis.
2. Se o doador for anônimo, respeitar nome público.
3. Carteirinha não deve dizer “ativo” se assinatura estiver cancelada/inadimplente, salvo regra definida.
4. Pix único confirmado pode gerar reconhecimento, mas não deve ser confundido com assinatura mensal ativa.
5. Não implementar Apple Wallet/Google Wallet nesta fase.
6. Não expor dados financeiros na página pública da carteirinha.

## O que não implementar agora

Não implementar Apple Wallet/Google Wallet, impressão física, mural completo ou sistema avançado de selos.

## Segurança obrigatória

1. Validar ownership da carteirinha.
2. Não expor dados sensíveis na rota pública.
3. Assinar ou gerar segredo interno para validação.
4. Registrar geração e alterações em auditoria, se aplicável.
5. Evitar enumeração massiva de carteirinhas públicas.
6. Não retornar CPF, telefone, endereço ou e-mail na validação pública.

## Testes obrigatórios

1. Gerar carteirinha.
2. QR Code abre página pública.
3. Doador anônimo não expõe nome completo.
4. Download PNG/PDF funciona.
5. Doador não acessa carteirinha de outro.
6. Status cancelado/inativo aparece corretamente.
7. Layout mobile.
8. Dados são gravados e lidos do MySQL corretamente.

## Critérios de aceite

1. Carteirinha aparece corretamente.
2. QR Code valida publicamente sem dados sensíveis.
3. Downloads funcionam.
4. Build e testes passam.
5. Privacidade do doador é preservada.
