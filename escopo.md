Escopo do Site/PWA — Lar dos Anjos Pet
1. Objetivo central do projeto

Criar uma plataforma profissional para transformar visitantes em apoiadores recorrentes do abrigo, com foco em:

Doações mensais a partir de R$ 9,90.
Doações únicas por Pix, cartão, boleto e demais meios suportados pelo Asaas.
Pix recorrente / Pix Automático, quando disponível e habilitado na conta Asaas.
Portal de transparência com arrecadação, despesas, custo diário e prestação de contas.
Área exclusiva para doadores, com carteirinha digital, histórico de impacto e benefícios simbólicos.
Painel administrativo completo para gestão de doadores, pagamentos, animais, despesas, campanhas e conteúdos.
Arquitetura preparada para PWA agora e aplicativo nativo futuramente.

A API do Asaas permite automatizar cobranças e recebimentos com meios como Pix, boleto, cartão de crédito, cartão de débito e TED. O fluxo-base recomendado é criar um cliente no Asaas, criar cobranças ou assinaturas vinculadas a esse cliente e manter o sistema sincronizado por webhooks.

2. Posicionamento emocional e estratégico

O site não deve parecer apenas uma “página de doação”. Ele deve fazer a pessoa sentir que está entrando para uma comunidade de proteção animal.

Mensagem principal:

Seja um Anjo mensal e ajude a manter vidas protegidas todos os dias.

Tom da marca:

Afetivo, mas profissional.
Transparente, sem sensacionalismo.
Esperançoso, sem exploração emocional excessiva.
Direto, com impacto claro: “sua doação ajuda em ração, medicamentos, castrações, limpeza, transporte e cuidados diários”.

Gatilhos mentais saudáveis:

Pertencimento: “Você faz parte da rede de Anjos”.
Identidade: “Sou um protetor dos animais”.
Compromisso contínuo: assinatura mensal simples e acessível.
Prova social com consentimento: ranking ou mural apenas para quem autorizar.
Transparência: mostrar para onde vai o dinheiro.
Impacto tangível: “sua contribuição ajudou em X kg de ração, Y atendimentos, Z resgates”.
Reconhecimento simbólico: carteirinha, selos, certificados e mural dos apoiadores.
Baixa barreira de entrada: planos desde R$ 9,90.

14. Integração com Asaas
Modelo recomendado

O frontend nunca deve conversar diretamente com a API do Asaas. O fluxo correto é:

Usuário escolhe plano ou valor.
Frontend envia dados para o backend.
Backend cria/atualiza cliente no Asaas.
Backend cria cobrança única ou assinatura.
Backend salva o registro local.
Usuário paga.
Asaas envia webhook.
Backend valida e atualiza status.
Área do doador e transparência são atualizadas.
Entidades locais
asaas_customer_id
asaas_payment_id
asaas_subscription_id
asaas_webhook_event_id
payment_status
billing_type
due_date
confirmed_at
received_at
Doação única

Fluxo:

Criar cliente.
Criar cobrança.
Retornar link/QR Code/dados de pagamento.
Aguardar confirmação por webhook.
Atualizar doador e transparência.
Doação mensal

Fluxo:

Criar cliente.
Criar assinatura mensal.
Salvar assinatura local.
Monitorar cobranças geradas.
Atualizar status por webhook de cobrança.
Pix recorrente

Implementar em duas camadas:

Pix Automático, quando a conta Asaas e a API estiverem habilitadas.
Fallback Pix mensal, gerando cobrança recorrente com lembretes automáticos.
Webhooks obrigatórios

O sistema deve lidar com:

Pagamento criado.
Pagamento confirmado.
Pagamento recebido.
Pagamento vencido.
Pagamento cancelado.
Pagamento estornado.
Pagamento deletado.
Falha de pagamento.
Atualização de cobrança.

Boas práticas:

Idempotência: o mesmo webhook pode chegar mais de uma vez.
Fila de processamento.
Log completo.
Reprocessamento manual.
Não confiar apenas no redirect do checkout.
Alertar admin quando webhooks falharem.

Preenche dados básicos.
Escolhe Pix/cartão/boleto.
Backend cria cobrança.
Usuário paga.
Webhook confirma.
Doação aparece na transparência.
Doador recebe agradecimento.
Fluxo de transparência
Admin lança despesa.
Anexa comprovante.
Define se será público.
Sistema atualiza gráficos.
Página pública exibe lançamento.
Relatório mensal consolida dados.
Fluxo de inadimplência
Pagamento vence ou falha.
Webhook atualiza status.
Sistema marca assinatura como pendente/inadimplente.
Doador recebe aviso amigável.
Área do doador mostra botão de regularização.
Após pagamento, status volta para ativo.
